# Contributing

Code and public knowledge contributions are welcome through Pull Requests.
Direct changes to `main` are not part of the contribution workflow.

## Code Changes

1. Add or update tests first.
2. Run `python -m pytest -q`.
3. Run `python scripts/release_check.py --root .`.
4. Explain behavioral and data-contract changes in the Pull Request.

## Knowledge Changes

- Use original explanations and include source references.
- Keep canonical knowledge independent from any one textbook edition.
- Record textbook facts only as short location metadata.
- Do not include textbook prose, exercises, answers, illustrations, scans,
  teacher-guide content, or continuous restorable excerpts.
- Do not include student information, homework, photos, mistakes, mastery data,
  or family observations.
- A model's confidence is not evidence. State uncertain claims explicitly.

All merged public knowledge remains subject to a second local rule-and-model
review before it enters a user's formal local knowledge base.
