# Security Policy

## Supported Version

Security fixes are applied to the latest release on `main`.

## Reporting

Do not open a public issue containing credentials, student information, private
Vault contents, or unpublished textbook material. Use GitHub's private security
advisory feature for sensitive reports.

## Data Boundary

The public project never needs student names, homework images, school details,
mastery records, or private Vault files. A report containing those materials
will be removed from public discussion.
